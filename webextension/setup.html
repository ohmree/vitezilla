            # Setup

            # Requirements

            You need to be using webextension-polyfillopen in new window and @types/webextension-polyfillopen in new window in your webextension for this project to work.

            If you don't want to use the above in your webextension, you might want to take a look at vitezillaopen in new window instead and find a custom solution. Take a look at the deep-mock documentation to get an idea.

            $ pnpm i -D @vitezilla/webextension

            $ yarn add -D @vitezilla/webextension

            $ npm i -D @vitezilla/webextension

            # Vitest

            Aside from vitezilla and @vitezilla/webextension themselves, you'll only need Vitestopen in new window.

            Here's a quick setup guide for Vitest with TypeScript and @vitezilla/webextension:

            $ pnpm i -D vitest

            $ yarn add -D vitest

            $ npm i -D vitest

            Create a file vitest.config.ts or append to your existing vite.config.ts file:

            /// <reference types="vitest" />

import { defineConfig } from 'vite';

export default defineConfig({
  test: {
    setupFiles: 'vitest.setup.ts',
  },
});

            Tips

            You might need to use Vite's mergeConfigopen in new window helper if you create a separate config for Vitest and would like to share some of the options between the two.

            Add test scripts to your package.json:

            {
  "scripts": {
    "test": "vitest run",
    "test:watch": "vitest watch"
  }
}

            In order to setup afterEach callbacks and mock webextension-polyfill you'll need to create a setup file (e.g. vitest.setup.ts):

            import '@vitezilla/webextension/setup';

            Now all you need to do is write test files and run the tests:

            $ pnpm test

            $ yarn test

            $ npm test
